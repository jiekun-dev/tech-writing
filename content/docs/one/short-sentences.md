# 1.4 短句
> 预计阅读时间：20 分钟

软件工程师们通常都会尽量精简一个功能实现的代码量，原因如下：
- 更短的代码更易于阅读。
- 更短的代码更易于维护。
- 每多加一行代码，就会多引入一个出错的地方。

实际上，技术写作业符合这些规律：
- 更短的文档更易于阅读。
- 更短的文档更易于维护。
- 每多写一行文字，就多一处可能出错的地方。

找到最短的写作路径需要花些时间，但这是百分百值得的。短句比长句在表达上更有力量，而且通常更容易理解。
